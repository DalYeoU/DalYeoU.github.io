팀스파르타 Unreal 8기 CH2 1번 과제

오늘은 강의들을 보고 공부하며 미뤄둿던 1번 과제를 마무리했다. 과제 답을 적을순 없지만 과제를 진행하면서 배운 내용들을 정리해 볼까 한다.

과제를 진행하며 배운 내용

  • 포인터: &로 주소를 찾아가고, *로 해당 주소의값을 수정하는 포인터의 원리.
  • 모듈화: 기능을 별도의 함수와 별도의 파일(.h,.cpp)로 나누어 관리하는 설계 방법
  • 최적화: * 2대신 비트 시프트 연산자(<< 1)를 사용하여 컴퓨터의 연산 능력을 향상시키는 방법
  • 예외처리: std::cin.fail()
    • 을 활용해서 입력 오류를 처리하는 방법

1-1. 메모리와 주소

{% highlight cpp %} void setPotion(int count, int* p_HPPotion, int* p_MPPotion) { /* 빈칸 2: 포인터가 가리키는 값에 접근 */ p_HPPotion = count; /* 빈칸 3: 포인터가 가리키는 값에 접근 */ p_MPPotion = count; } {% endhighlight %}

위 코드는 과제에서 제공해주는 힌트의 일부이다. 위 예제를 보면 int* p_HPPotion 이 부분에 포인터 선언이 되어있다.

이건 일반 숫자를 담는 바구니가 아닌 주소값만 담을 수 있는 바구니이다.

그렇기에 이 변수는 일반 숫자나 변수 이름을 그냥 주면 안되고 반드시 주소값을 담아야 한다.

1-2. 주소를 만드는 방법

여기서 우리는 마법의 지팡이(&)로 주소를 만들어줘야 한다.

만약 player.hpPotion이라고만 변수를 줬다고 가정해보면 위 예제에는 당연하게도 변수의 값이 넘어가기에 작동하지 않을거다.

하지만 player.hpPotion앞에 &를 붙히게 된다면 변수의 값이 아닌 주소값이 된다.

그렇다면 위 함수에서 요구하는 주소를 넘겨주기에는 충족되게 된다.

하지만 이건 위 함수를 호출할때 필요한 부분이고 해당 힌트에서 필요한 부분은 아니다.

1-3. 주소를 활용하는 방법

이제 위 예제를 채우기 위해 우리에게 필요한건 주소를 따라가게 만드는 마법사의 돌(*)이다

이미 2번에서 주소를 만드는 방법을 알아보았다. 그럼 그 주소를 사용할 방법이 있어야한다.

해당 함수를 호출할때 &player.hpPotion이렇게 작성했다고 가정해보자.

여기서 주소를 받아왔기 때문에 주소를 따라가야 한다. 포인터는 &로 주소를 저장하고 *로 해당 주소의 값을 따라간다.

그럼 이제 위 예제의 답을 채워넣을수 있게되었다.

며칠동안 포인터와 레퍼런스 부분을 찾아보고 강의를 보고 예제코드를 뜯어보며 공부해보았는데 과제로 사용하게 되니 확실이 더 이해가 쉬웠다.

2. 모듈화

해당 부분은 필자가 과거 파이썬과 자바를 다룰때도 자주 사용해봤던 부분이다.

메인 구동 함수에 모든 코드를 집어넣지 말고 기능마다 함수로 분리해서 개별로 관리하고 필요하면 파일을 나눠서 파일별로도 나눠서 관리하는 방법이다.

예를 들어 메인함수에 시작하자마자 변수를 초기화하고 게임을 시작하고 캐릭터를 불러오고 없으면 없으면 캐릭터를 만들고 게임을 시작하고......

이렇게 되면 코드가 굉장히 보기 불편해지고 알아보기도 어려울것이다. 개발을 혼자서 한다면 혼자만 불편하면 끝이지만 팀단위라면...

그래서 필자는 해당 과제를 해결할때 파일을 총 3개로 나눴다. main.cpp, Character.cpp, Character.h 이렇게 3개로 나눴다.

그리고 파일을 나누며 각각의 기능들을 전부 하나의 함수로 분리해서 개별로 관리를 하고있다.

Character.h는 설계도 부분이다. 필요한 함수들을 이곳에서 선언한다. 식당으로 비유하면 이곳은 식당입구의 메뉴판이다.

Character.cpp는 실제 구현부이다. 위 설계도에 적힌 부분을 이곳에서 실제로 구현한다. 이것도 식당으로 비유하면 주문이 들어온 요리를 하는 주방이다.

마지막으로 main.cpp는 지휘소이다. 프로그램의 시작부분으로 매우 중요한 부분이다. 이부분은 식당의 홀 정도로 비유할수 있겠다.

3. 최적화

사실 이부분에 대해서는 말할수 있는 부분이 많이 있지는 않다.

과제에서 특정 능력치를 두배 증폭 시키라는 부분이 있는데 값을 두배로 만드는 방법은 두가지가 존재한다.

단순하게 2를 곱하거나(* 2), 비트 시프트 연산자(<< 1)를 사용할수 있다.

두개의 차이는 코드의 가독성과 처리 속도에 있다.

단순하게 곱하기 연산을 하게되면 코드의 가독성은 좋지만 처리 속도는 일반적인 속도다.

하지만 비트 시프트 연산자를 사용하게 되면 코드의 가독성은 떨어지지만 처리속도는 훨씬 빠르다.

4. 예외처리

이부분 또한 말할수 있는 부분이 많이 없다.

우리가 과제로 만든 게임은 콘솔창으로 하는 텍스트 게임이다.

그렇기에 콘솔창에 유저가 직접 값이나 선택을 입력하는 방식으로 진행하게 되는데 그부분에서 다른 문자를 작성했을때 오류가 나는걸 방지했다.

만약 유저가 무언가 작성할때 숫자만 입력해야 하는데 문자까지 작성했다고 가정해보자.

그렇게 됐을때 입력 버퍼의 문제로 에러가 발생해서 무한루프에 빠지게 된다.

그 상태를 해결하려면 잘못된 입력을 비워주는 과정이 필요하다.
