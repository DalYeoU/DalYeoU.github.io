오늘 학습한 내용 정리

블루프린트란?

블루프린트란 언리얼 엔진에서 제공하는 시각적인 프로그래밍 기법입니다.

코드를 작성하는 대신 시각적인 인터페이스를 통해 게임 로직을 구현할 수 있도록 도와주는 시스템입니다.

현업에서는 프로토타입 구현 혹은 아트 및 기획자 팀원들이 컨텐츠 구현시 사용한다고 합니다.

레벨 블루프린트 실습

이벤트 함수: 어떤 사건(Event)이 발생하면 호출되는 함수.

  • BeginPlay : “게임 플레이의 시작”이라는 사건이 발생하면 호출되는 함수
  • Tick: “매 게임 프레임”마다 호출되는 함수

레벨 블루프린트에서 이벤트 함수들을 활용해 게임이 시작될 때마다 로그가 출력되도록 구현해보았습니다.

BeginPlay 노드는 위와 같이 시작하면 호출되는 함수이다. 이걸 이용해서 게임 시작 시 특정 로그를 출력할 수 있습니다.

다음으로 Tick 이벤트 함수를 사용해 매 프레임마다 로그를 출력해보았습니다.

Tick 함수는 게임의 매 프레임마다 호출되는 함수입니다. 이 함수를 활용해 반복적인 작업을 구현할 수 있습니다.

추가로 Branch와 Sequence 라는 노드를 사용해 조건에 따라 다른 작업을 수행할 수 있습니다.

Branch와 Sequence

  • Branch : 조건에 따라 다른 경로로 실행 흐름을 나눠주는 노드
  • Sequence : 순차적으로 실행되는 노드

branch 노드는 바로 위 Tick 함수 예시에서 사용되었고 Sequence 또한 마찬가지로 Tick 함수에서도 사용되었습니다.

바로 위 Sequence 노드 사용 예시는 Branch를 사용한 예시와 동일한 구조를 가지고 있지만 옆으로 길게 늘어진 노드들을 정리하고 가독성을 높일 수 있습니다.

반복문 사용 예시

Tick 함수 사용 예시를 활용해서 반복문을 사용해보았습니다.

For Loop와 While Loop

  • For Loop : 특정 횟수만큼 반복하는 노드
  • While Loop : 조건이 참인 동안 반복하는 노드

반복문이지만 for문은 특정 횟수만큼만 반복하는 노드이고 While 문은 횟수를 정해놓지 않고 조건이 만족 될때까지 반복하는 노드입니다.

반복문이라는 공통점이 있지만 for 문과 While문은 반복하는 방식이 다르기 때문에 상황에 맞게 적절히 사용해야할 것 같습니다.

또한 지금까지 추가로 적지는 않았지만 지금까지의 예시들에 변수들이 들어가있다.

변수의 종류

  • Boolean논리 : 참True과 거짓False을 나타내는 변수
  • Byte정수 : 가장 크기가 작음(0~255)
  • Integer정수 : 일반적인 정수(-21억~21억)
  • Integer64정수 : 가장 크기가 큼
  • Float실수 : 정밀도가 더 우월함(double-precision)
  • Name문자열 : 엔진 내부에서 사용, 불변인 경우
  • String문자열 : 일반적인 문자열 (ex. 플레이어명)
  • Text문자열 : 퀘스트 설명같이 다국어 변환이 필요한 경우

변수는 데이터를 저장하는 공간입니다. 각 변수는 특정한 값을 가질 수 있고 그 값은 프로그램이 실행중이라고 하더라도 얼마든지 변경이 가능합니다.

텍스트 슈팅 게임 만들기

지금까지 배운 블루프린트 기능을 이용하여 간단한 텍스트 슈팅 게임의 주요 기능인 총기 기능을 구현해 보았습니다.

총기 기능

  • 발사 : 총알이 발사되는 기능
  • 재장전 : 탄창이 비었을 때 탄창을 교체하는 기능
  • 탄약 관리 : 현재 남아있는 탄약 수를 관리하는 기능
  • 과열 기능 : 총기를 너무 자주 사용할 경우 과열이 발생하여 발사가 불가능해지는 기능

총기 종류

이름 탄창
Rifle 30
Shotgun 8
Revolver 6

위 표는 이번에 구현한 총기들의 종류와 각 총기의 탄창 수입니다. 각 총기는 고유한 탄창 수를 가지고 있으며, 발사 시마다 탄약이 감소하게 됩니다.

과열 기능은 임시로 총알 한발당 과열게이지 1씩 증가하게 되고 게이지가 30까지는 Normal상태, 31~70까지는 High상태, 71~100까지는 Over상태입니다.

과열 상태에 따른 패널티는 아직 구현하지 않았지만 100이되어 OverHeat상태가 되면 총기를 사용할 수 없게 됩니다.

과열 상태는 임시로 특정 키를 누르면 1씩 감소하는 식으로 구현해두었습니다.

총기 인식 -> 과열 상태 인식 -> 탄약 상태 확인 -> 격발 구조로 구현되어있습니다.

먼저 총기를 인식하는 노드를 사용하여 현재 들고 있는 총기를 확인합니다. 이 노드는 총기의 종류에 따라 다른 동작을 하도록 설계되어 있습니다.

들고 있는 총기가 라이플일 경우에는 좌클릭을 누르고 있는 동안에 총이 격발됩니다. 만약 샷건이나 리볼버일 경우에는 클릭 한 번으로 단발 발사가 됩니다.

격발이 되면 과열 게이지를 증가시키고 총알을 감소시킵니다.

옆에 우측의 사진은 과열 상태를 증가시키는 함수입니다. 간단하게 0~30까지는 Normal상태, 31~70까지는 High상태, 71~100까지는 Over상태로 변환시켜주는 함수입니다.

격발이 되면 과열 게이지가 증가하고 탄약이 감소합니다. 과열 상태에 따라 총기의 사용이 제한되며, 특정 키를 통해 과열 게이지를 감소 시킬 수 있습니다.

또한 재장전 기능은 탄약이 다 떨어졌을때는 재장전이 필요하다고 표시되며 탄약이 이미 최대치일경우 재장전이 필요하지 않다는 로그를 출력합니다.

또한 사진은 없지만 총기 쿨링다운 기능은 0 이하로 내려가지 않도록 구현되어 있습니다.

학습하며 겪은 문제점

기능 구현을 하는 도중에 알수없는 이유로 총기 기능이 고장이 난적이 있었습니다.

총을 격발하게 되면 첫번째 총알은 발사가 되지만 두번째 총알부터는 발사가 되지 않았습니다.

이 문제는 총기의 탄약 수를 감소시키는 로직이 잘못되어 있었기 때문에 발생했습니다.

총알이 발사될 때마다 탄약 수가 감소해야 하는데, 탄약 수를 감소시키는 노드가 잘못 연결되어 있었고, 탄약 수를 감소시키는 로직 자체도 잘못 구현되어 있었습니다.

이 문제가 발생했을때 정말 어디서부터 잘못된건지 찾는 데 많은 시간을 소비했습니다.

결과적으로 Set Array Element 노드에 탄약을 갱신시키는 핀이 연결이 끊어져있는걸 발견했고 이부분을 수정하니 다시 정상적으로 작동했습니다.

이 강의를 시작하기 전에는 블루프린트는 시각적으로 잘 보이기 때문에 코드보다 디버깅이 쉬울 것 같았지만 실제로는 그와 비슷한거 같다는 생각이 들었습니다.

하지만 이정도는 필자가 아직 블루프린트에 익숙하지 않아서 생긴 해프닝이라고 생각합니다.

시간이 더 있었다면 몇가지를 더 만들어보고 싶었지만 시간이 너무 늦어 그정도까지는 하지 못했습니다.

아쉽지만 다음에 시간이 있다면 더 많은 기능들을 구현해보고 싶습니다.
