오늘 학습한 내용 정리

C++에 대해 학습한 내용 정리

오늘은 포인터에 대해서, 그리고 레퍼런스에 대해서 학습했습니다.

1. 레퍼런스

레퍼런스는 포인터와 비슷하지만 약간 다르다. 간단하게 말하자면 레퍼런스는 변수에 별명을 부여하는 기능을 합니다.

레퍼런스는 선언과 동시에 초기화가 되어야 하며, 한번 초기화된 레퍼런스는 다른 변수에 참조할 수 없습니다. 또한 레퍼런스는 포인터와 달리 null이 될 수 없습니다.

간단한 예제

{% highlight cpp %} #include using namespace std; void addTenRef(int& r) { r = r + 10; cout << "레퍼런스 사용:" << r << endl; } int main() { int num = 5; addTenRef(num); } {% endhighlight %}

실행 결과:

{% highlight text %} 레퍼런스 사용: 15 {% endhighlight %}

위 예제를 보면 레퍼런스는 단순하게 기존의 변수처럼 사용이 가능합니다. 호출하기도 편하고 해당 변수에 직접 접근할 수 있습니다.

2. 포인터

포인터는 저번 TIL에서도 있는 내용입니다. 간단하게 말해서 레퍼런스는 해당 변수에 별명을 부여하는 기능이지만 포인터는 변수의 주소를 저장하는 기능을 합니다.

간단한 예제

{% highlight cpp %} #include using namespace std; void addTenPointer(int* p) { *p = *p + 10; cout << "포인터 사용:" << *p << endl; } int main() { int num = 5; addTenPointer(&num); } {% endhighlight %}

실행 결과:

{% highlight text %} 포인터 사용: 15 {% endhighlight %}

위 예제를 보면 포인터는 해당 변수의 주소를 저장하기 때문에 호출할 때 주소값을 전달해야 합니다. 그리고 포인터가 가리키는 값을 변경하려면 *p와 같이 역참조 연산자를 사용해야 합니다.

3. 포인터와 레퍼런스의 차이점

포인터와 레퍼런스는 모두 변수에 대한 간접적인 접근을 제공하지만 차이점이 있습니다.

포인터와 레퍼런스의 차이

구분 포인터(int* p = &a;) 레퍼런스(int& r = a;)
특징 변수의 주소를 저장 변수에 대한 별명
접근 방법 *p로 값 접근 변수 이름처럼 접근
초기화 나중에 주소 저장 가능(nullptr가능) 선언과 동시에 초기화 필요
재할당 다른 주소로 재할당 가능 한 번 초기화되면 다른 변수 참조 불가

위 표를 보면 포인터는 사용이 유연하지만, 레퍼런스는 더 간단하고 안전한 방식으로 변수를 참조할 수 있습니다.

학습하며 든 궁금증

그럼 무조건 레퍼런스가 좋은 거 아냐? 포인터는 왜 써?라는 의문점이 들었다.

포인터와 레퍼런스의 차이점은 유연성이 있고 안전성이 있습니다.

예를들어, 포인터는 이번엔 A의 주소를 저장했다가 바로 다음엔 B의 주소를 저장하는 방식이 가능하지만, 레퍼런스는 그게 안된다. 한번 별명을 지정하면 변경이 안되기 때문이다.

하지만 레퍼런스는 포인터보다 사용하기 훨씬 편하기에 원본을 안전하고 깔끔하게 수정할때 사용하기에 좋다.
